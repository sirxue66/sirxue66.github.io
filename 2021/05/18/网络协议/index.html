web
  • Net

DNS解析（Domain Name Server）【域名服务器】

作用

域名 与 IP 转换的服务器

特点

DNS中保存一种域名与ip对应的表
一个域名只能对应一个ip地址，一个ip地址可以对应多个域名

过程

 浏览器，输入域名  -->  本地DNS服务器（其实就是服务商提供的DNS服务器，连网），如果本地DNS服务器没有缓存，就会访问远程DNS服务器寻找ip地址。
                    -->   从其他远程DNS服务器拿到ip地址后，会将ip地址发给浏览器，并写入缓存。

IP地址：【互联网协议 地址】

作用

分配给用户上网用的互联网协议

分类

IPv4  -->  32位，4组 十进制数       192.168.101.129   
IPv6  -->  12位，8组  十六进制      ABCD.EF01.2345.6789.ABCD.EF01.2345.6789

端口号

作用

每一个端口对应于一个服务器下的一个业务或者项目，访问同一个ip地址的不同端口号，其实就是访问同一台服务器中的不同服务。

范围

2 - 65535

默认端口号

http协议下（IPv4）： 80
https协议下（IPv6）： 443

TCP(Transtraction Control Protocol)传输控制协议

特点

面向连接（收发数据前，都会建立可靠的连接）

建立连接的基础

三次握手

优点

稳定、重传机制、安全

缺点

速度慢、占用资源

TCP/IP协议组

提供了点对点的连接机制

UDP(User Data Protocol)用户数据报协议

特点

面向无连接，不可靠的协议，无状态传输机制

应用场景

无需确保通讯质量且要求快，比如直播、语音通话

优点

快速、安全

缺点

不稳定、不可靠 、丢包

HTTP 与 HTTPS

HTTP(超文本传输协议)

定义：
        客户端与服务器间请求与应答的标准，用于从web服务器传输超文本到本地浏览器的传输协议。（传输的明文）
HTTP请求：
        按照协议规定先向web服务器发送的将超文本传输到本地浏览器的去请求

HTTPS(超文本传输安全协议)

定义：
        HTTP的安全版（对传输数据使用SSL和TLS进行了加密，传输的是密文）

TCP连接 三次握手

如图：

第一次

客户端向服务器发送SYN数据包(标志位)，并进入  等待服务器确认状态

第二次

服务器收到来自客户端的SYN，服务器确认该数据包，并发送 ACK 和 (SYN+1)回去，服务器进入请求接收并等待客户端确认状态。

第三次

客户端进入链接状态后，向服务器发送(ACK+1)和(SYN+1)表示建立连接确认，服务器收到确认后，进入链接状态。
