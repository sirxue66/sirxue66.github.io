网络协议

DNS解析（Domain Name Server）【域名服务器】

作用

域名 与 IP 转换的服务器

特点

DNS中保存一种域名与ip对应的表
一个域名只能对应一个ip地址，一个ip地址可以对应多个域名

过程

 浏览器，输入域名  -->  本地DNS服务器（其实就是服务商提供的DNS服务器，连网），如果本地DNS服务器没有缓存，就会访问远程DNS服务器寻找ip地址。
                    -->   从其他远程DNS服务器拿到ip地址后，会将ip地址发给浏览器，并写入缓存。

IP地址：【互联网协议 地址】

作用

分配给用户上网用的互联网协议

uni-App

什么是 uni-App

uni-app 是一个使用 Vue.js 开发所有前端应用的框架，开发者编写一套代码，可发布到iOS、Android、Web（响应式）、以及各种小程序（微信/支付宝/百度/头条/QQ/钉钉/淘宝）、快应用等多个平台。
官网： https://uniapp.dcloud.io

框架结构

pages.json

pages.json 文件用来对 uni-app 进行全局配置，决定页面文件的路径、窗口样式、原生的导航栏、底部的原生tabbar 等。

manifest.json

manifest.json 文件是应用的配置文件，用于指定应用的名称、图标、权限等。HBuilderX 创建的工程此文件在根目录，CLI 创建的工程此文件在 src 目录。

package.json

通过在package.json文件中增加uni-app扩展节点，可实现自定义条件编译平台（如钉钉小程序、微信服务号等平台）。

vue.config.js

vue.config.js 是一个可选的配置文件，如果项目的根目录中存在这个文件，那么它会被自动加载，一般用于配置 webpack 等编译选项.

git使用

git的使用

git 是什么？

git就是： 代码的版本管理工具
git工作流程： 工作区 –git add –> 暂存区 –git commit–> 版本区

配置：

1，配置全局用户名： git config –global user.name ‘ ‘
2,配置全局邮箱： git config –global user.email ‘ ‘
3,查看配置信息： git config –list

增加 工作区—->暂存区

1,增加所有更新文件： git add .
2,增加指定文件： git add 文件名

提交 暂存区—–>版本区

git commit -m “说明”

查看

1,查看状态（工作区与暂存区之间的状态）： git status
2,查看提交记录（暂存区提交到版本区的提交记录）：git log
